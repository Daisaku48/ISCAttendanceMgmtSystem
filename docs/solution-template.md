# ソリューションテンプレート化ガイド

このドキュメントは、既存の 3 層構成を基準にした Web アプリテンプレートを作成するための整理ポイントとチェックリストをまとめています。

## 1. プロジェクト構成（推奨）
- **COM**: 共通モデル、DTO、ユーティリティ
- **BL**: ビジネスロジック、サービス、ドメイン規約
- **Web**: MVC/UI、ViewModel、Controller

## 2. テンプレート化の方針
- **構造は維持**: 現行構造を崩さず再利用性を上げる
- **置換ポイントを明示**: プロジェクト名・名前空間・接続文字列
- **最小限の差分**: 新規案件ごとの差分を減らす

## 3. 置換ポイント一覧
| 種別 | 例 | 作業内容 |
| --- | --- | --- |
| ソリューション名 | `ISCAttendanceMgmtSystem.sln` | 新プロジェクト名に変更 |
| プロジェクト名 | `ISCAttendanceMgmtSystem.Web` | フォルダ・csproj名を変更 |
| 名前空間 | `ISCAttendanceMgmtSystem` | 全プロジェクトで置換 |
| 設定値 | `Web.config` | 接続文字列・AppSettings の更新 |

## 4. 新規案件用チェックリスト
- [ ] ソリューション名を変更
- [ ] COM/BL/Web のプロジェクト名を変更
- [ ] 名前空間の置換
- [ ] 接続文字列・設定値を更新
- [ ] 参照パスが壊れていないか確認
- [ ] ビルド/起動確認

## 5. 運用ルール（提案）
- 命名規約は `Company.Product.*` を基本にする
- 共通ロジックは COM に集約し、BL でユースケース化
- Web 層は UI に限定し、BL 以外への直接依存は避ける
